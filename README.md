# Patchwork - Instructions

## Prerequisites

### Node.js

- Node.js 18+ and npm

### Python (optional, for full baseline analysis)

- Python 3.8+
- Install: `pip install mypy bandit semgrep pycg`

### API Keys

Create a `.env` file in the project root:

```
OPENAI_API_KEY=sk-...           # Required for GPT-4o evaluations
ANTHROPIC_API_KEY=sk-ant-...    # Required for Claude evaluations
HUGGING_FACE_HUB_TOKEN=hf_...  # Required for HuggingFace/Qwen evaluations
GOOGLE_API_KEY=AIza...          # Optional
```

Not all keys are needed for every run — only the key for the model you're evaluating.

## Setup

```bash
npm install
npm run build
```

Verify the build works:

```bash
npm test
```

## Running the Evaluation Pipeline

The project provides several entry points, from lightweight verification to full-scale evaluations.

### 1. Dry Run (no API keys needed)

Validates the entire pipeline end-to-end using 3 synthetic repositories with hand-crafted code containing intentional structural failures. No LLM calls are made.

```bash
npx ts-node src/dry-run.ts
```

**What it does:**
- Creates 3 synthetic repos (FastAPI, Django, Express) with planted bugs
- Runs graph construction (8 graph types) on each
- Runs failure detection (8 categories: PIA, DHI, BCI, SRF, RCF, CFC, CCV, SSR)
- Calculates metrics (precision, recall, F1)
- Prints a pipeline checklist showing which graph types and failure categories were detected
- Saves results to `outputs/dry_run_<timestamp>.json`

**Expected output:** `ALL CHECKS PASSED` at the end if the pipeline is working correctly.

### 2. Phase 2 - Scaled Pilot (Cloud LLMs)

Runs 10 curated repos (5 Python + 5 TypeScript), 6 tasks each, through a single model with P1 prompting. Requires either `OPENAI_API_KEY` or `ANTHROPIC_API_KEY`.

```bash
# Full run with Claude (60 tasks)
npx ts-node src/run-phase2.ts

# Full run with GPT-4o
npx ts-node src/run-phase2.ts --model=gpt4o

# Smoke test (1 repo, 1 task — verify API connectivity)
npx ts-node src/run-phase2.ts --dry
```

**Features:**
- Checkpoint/resume: automatically saves progress to `outputs/phase2_<model>_checkpoint.json` and resumes on restart
- Rate limiting: 1s delay between API calls
- Retry with exponential backoff (3 attempts)

**Output:** `outputs/phase2_<model>_<timestamp>.json`

### 3. HuggingFace Evaluation (Qwen2.5-Coder-32B)

Evaluates the open-source Qwen2.5-Coder-32B-Instruct model via HuggingFace Inference API. Same repos and tasks as the frontier evaluation for direct comparison.

```bash
# Full run (10 repos, 60 tasks)
npx ts-node src/run-hf-evaluation.ts

# Smoke test (1 repo, 1 task)
npx ts-node src/run-hf-evaluation.ts --dry

# Limit total tasks
npx ts-node src/run-hf-evaluation.ts --limit=5
```

**Requires:** `HUGGING_FACE_HUB_TOKEN` in `.env`

**Output:** `outputs/hf_eval_<timestamp>.json`

### 4. Local GPU Model Evaluation

Evaluates local GPU-hosted models (Qwen-7B, StarCoder2-15B, optionally CodeLlama-34B) via an HTTP API on a GPU machine.

```bash
npx ts-node src/run-local-evaluation.ts
```

**Prerequisites:** A running model server at `http://10.96.50.180:8000` (configured in `src/run-local-evaluation.ts`). Edit the URL to match your setup.

**Output:** `outputs/local_models_<timestamp>.json`

### 5. BaxBench Evaluation (Django)

Evaluates Django code generation from BaxBench benchmark tasks.

```bash
# Prerequisite: export BaxBench tasks
python3 scripts/run-baxbench.py

# Run with GPT-4o (default)
npx ts-node src/run-baxbench-eval.ts

# Run with Claude
npx ts-node src/run-baxbench-eval.ts --model=claude

# Run with Qwen
npx ts-node src/run-baxbench-eval.ts --model=qwen

# Smoke test
npx ts-node src/run-baxbench-eval.ts --dry

# Limit tasks
npx ts-node src/run-baxbench-eval.ts --limit=5
```

**Requires:** `outputs/baxbench_django_tasks.json` (generated by the Python script) and the appropriate API key for the selected model.

**Output:** `outputs/baxbench_eval_<model>_<timestamp>.json`

### 6. WebApp1K Evaluation

Runs the failure detection pipeline on WebApp1K benchmark samples (React/JSX single-file components).

```bash
# Default: 200 scenarios
npx ts-node src/run-webapp1k.ts

# Quick smoke test
npx ts-node src/run-webapp1k.ts --limit=2

# Full dataset
npx ts-node src/run-webapp1k.ts --limit=1000
```

**Requires:** `outputs/webapp1k_generations.json` (pre-generated code samples).

**Output:** `outputs/webapp1k_pipeline_results.json`


4. **Run the full Phase 2 evaluation:**
   ```bash

   ```

## Output

All evaluation scripts write their results as timestamped JSON files to the `outputs/` directory (created automatically on first run). Each output file includes the run configuration, stage timings, failure counts by category, and detailed results.

## Failure Categories Detected

| Code | Name | Description |
|------|------|-------------|
| PIA | Phantom Import/API | Imports or APIs that don't exist |
| DHI | Dependency Hallucination | Hallucinated or non-existent package dependencies |
| BCI | Build/Config Incoherence | Build or configuration problems (missing env vars, bad config) |
| SRF | Schema/Resource/Return Failures | Return type mismatches, schema violations |
| RCF | Resource Coherence Failures | Missing filesystem resources, return contract issues |
| CFC | Control Flow Coherence | Unreachable code, dead branches |
| CCV | Cross-file Contract Violations | Mismatched interfaces across files (field name mismatches) |
| SSR | Security Structural Regressions | Missing auth on destructive routes, security gaps |

```bash
npm run test:coverage       # Coverage report

# Single test file
npx jest src/__tests__/models.test.ts

# Tests matching a pattern
npx jest --testNamePattern="RepositoryModel"
```

Tests use Jest with fast-check for property-based testing (100 iterations, seed 42).
